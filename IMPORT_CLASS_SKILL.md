# 课程排课导入功能 - 前端开发指南

## 概述

本技能文档指导前端实现「课程排课 Excel 导入」功能，涉及学期管理、课程管理、教师管理、实验室选择及最终的数据导入。

导入流程：
1. 前置准备：确保学期、实验室、课程、教师基础数据已存在
2. 上传 Excel：按规范格式填写课表数据
3. 后端解析：逐条转换 → 成功写入数据库 / 失败回滚并返回错误明细
4. 前端展示：导入结果总览 + 逐条错误定位

---

## 接口清单

### 核心导入接口

```http
POST /academic/import
Content-Type: multipart/form-data
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `excel` | File | 是 | Excel 文件（.xls || .xlsx） |
| `semesterId` | Long | 是 | 学期 ID（下拉选择） |
| `laboratoryId` | Long | 是 | 实验室 ID（下拉选择） |

**响应结构：**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "ok": 10,
    "fail": 2,
    "errors": [
      {
        "rowIndex": 5,
        "columnIndex": 2,
        "rawContent": "软件工程<>1-16周[1-2节]<>A101",
        "reason": "单元格内容格式不符合要求，期望格式：课程<>时间<>地点<>教师"
      },
      {
        "rowIndex": 8,
        "columnIndex": 3,
        "rawContent": "高等数学<>1-8周[xx节]<>B202<>李四",
        "reason": "未找到任何匹配的时间信息"
      }
    ]
  }
}
```

| 字段 | 说明 |
|------|------|
| `ok` | 成功导入条数 |
| `fail` | 失败条数 |
| `errors[].rowIndex` | Excel 行索引（从 0 开始） |
| `errors[].columnIndex` | Excel 列索引（从 0 开始） |
| `errors[].rawContent` | 原始单元格内容 |
| `errors[].reason` | 失败原因 |

### 前置数据查询接口

导入前需要让用户选择学期和实验室，并了解已有课程/教师：

```http
GET /academic/list/semester          // 获取学期列表
GET /academic/list/laboratories      // 获取实验室列表
```

## Excel 格式规范

### 文件要求
- 格式：`.xlsx`
- 无表头（直接写数据）
- 每行对应一个时段，每列对应一个星期几（列 0=周一，列 1=周二... 或按实际排课习惯）

### 单元格内容格式

```
课程名称<>时间<>地点<>教师
```

使用半角 `<>` 作为分隔符（若 Excel 中输入全角符号，后端会自动清洗转换）。

### 时间字段格式

```
起始周-结束周[时间详情]
```

| 格式 | 示例 | 说明 |
|------|------|------|
| `X-Y周[节次]` | `1-16周[1-2节]` | 最常见，按节次排课 |
| `X-Y周(单)[节次]` | `1-16周(单)[3-4节]` | 单周上课 |
| `X-Y周(双)[节次]` | `1-16周(双)[5-6节]` | 双周上课 |
| `X-Y周[HH:mm-HH:mm]` | `1-8周[08:00-09:40]` | 按实际时间排课 |

### 一行多课程

同一单元格内有多门课程时，用**半角逗号 `,`** 分隔：

```
软件工程<>1-16周[1-2节]<>A101<>张三,高等数学<>1-8周[3-4节]<>A102<>李四
```

### 完整示例

假设 Excel 内容如下（列=星期几，行=时段）：

| | 周一(列0) | 周二(列1) | 周三(列2) |
|---|---|---|---|
| 第1-2节(行0) | `软件工程<>1-16周[1-2节]<>A101<>张三` | `高等数学<>1-8周[1-2节]<>B202<>李四` | |
| 第3-4节(行1) | | `大学英语<>1-16周[3-4节]<>C303<>王五` | `程序设计<>9-16周[3-4节]<>D404<>赵六` |

## 前端实现建议

### 1. 导入流程设计

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│ 1. 选择学期     │ --> │ 2. 选择实验室   │ --> │ 3. 上传 Excel   │
└─────────────────┘     └─────────────────┘     └─────────────────┘
                                                         |
                                                         v
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│ 6. 查看课表     | <-- │ 5. 查看错误明细 | <-- │ 4. 解析结果     │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 2. 文件上传组件

- 限制文件类型：`accept=".xlsx"`
- 上传前校验：非空、格式正确
- 建议提供「下载模板」功能，方便用户按规范填写

### 3. 结果展示面板

导入完成后展示三区域：

**总览卡片**
```
✅ 成功导入：{ok} 条
❌ 失败：{fail} 条
```

**成功提示**
- `fail === 0` 时显示绿色成功提示
- 提供「查看课表」跳转按钮

**错误列表（仅 fail > 0 时展示）**

| Excel 位置 | 原始内容 | 错误原因 |
|------------|----------|----------|
| 第 6 行，第 3 列 | `软件工程<>1-16周...` | 单元格内容格式不符合要求... |
| 第 9 行，第 4 列 | `高等数学<>1-8周...` | 未找到任何匹配的时间信息 |

> 行号展示建议：`rowIndex + 1`（因为后端返回的是从 0 开始的索引）

### 4. 错误处理建议

- **格式类错误**（如缺少 `<>`）：提示用户检查分隔符
- **时间类错误**（如 `[xx节]` 不合法）：提示用户时间格式应为 `X-Y节` 或 `HH:mm-HH:mm`
- **教师缺失**：当前实现允许空教师，不报错

## 数据模型关系

```
semester (学期)
  └── course_schedule (课表记录) [N]
        ├── course (课程) [1]
        ├── teacher (教师) [1]
        └── laboratory (实验室) [1]
```

导入操作本质上是：解析 Excel → 自动创建/复用 Course 和 Teacher → 生成 CourseSchedule 记录。

---

## 注意事项

1. **事务隔离**：每条数据独立事务，单条失败仅回滚该条，不影响其他数据
2. **数据重复**：当前实现中，每次导入都会创建新的 Course 和 Teacher 记录（即使名称相同）。若需去重，需提前通过 `/list/course` 和 `/list/teacher` 查询并引导用户选择已有数据
3. **全角符号**：Excel 中输入的全角数字、字母、空格会被后端自动清洗为半角，但 `<>` 分隔符和 `,` 分隔符建议用户直接输入半角
4. **列索引含义**：Excel 的列索引直接映射为星期几（如列 0=周一），行索引无特殊业务含义，仅用于错误定位
